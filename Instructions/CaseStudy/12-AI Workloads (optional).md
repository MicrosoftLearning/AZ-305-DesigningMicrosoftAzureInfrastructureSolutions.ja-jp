# ケース スタディ – ビジネス上の課題に対応する AI ツールを選択する

## グローバル小売組織向けの AI ワークロードの設計と実装

Tailwind Traders は、北米とヨーロッパで eコマースと実店舗の両方を運営する中規模のグローバル小売組織です。 この会社は急速な成長を遂げ、厳格なセキュリティ、ガバナンス、コンプライアンスの要件を満たしながら、カスタマー エクスペリエンスの向上、内部運用の合理化、在庫管理の最適化を実現するというプレッシャーが強くなっています。

経営陣は、AI のクラウド導入フレームワークに合わせた複数フェーズの AI 導入イニシアチブを承認しています。 目標は、時間の経過と共に、より高度な AI ワークロードのためのスケーラブルで管理された基盤を構築しながら、迅速な生産性の獲得を実現することです。

Tailwind Traders は、速さ、カスタマイズ、コスト、制御のバランスが取れた AI ソリューションを設計する必要があります。 組織は、サービスとしてのソフトウェア (SaaS)、サービスとしてのプラットフォーム (PaaS)、サービスとしてのインフラストラクチャ (IaaS) Azure AI オファリングを使用する混合型のアプローチを選択し、ビジネス上の問題ごとに適切なツールを選択します。

## ビジネス上の課題

Tailwind Traders は、いくつかの一般的なエンタープライズ AI の課題に直面しています。

1. 従業員は、情報の検索、ドキュメントの要約、日常的なコミュニケーションへの対応に過剰な時間を費やしています。

1. お客様は、eコマース サイト上でリアルタイムの会話形式の支援を期待していますが、現在のチャット ソリューションはルールベースで柔軟性がありません。

1. 在庫予測は、予測モデリングではなく履歴ルールに依存しており、過剰在庫や在庫が発生することになります。

## 要件

- [AI 戦略を作成する - クラウド導入フレームワーク](https://learn.microsoft.com/azure/cloud-adoption-framework/scenarios/ai/strategy) ガイダンスを確認します。
- 各ビジネス課題を Azure AI ソリューションに一致させます。
- 各選択肢について、その製品が最適な理由を簡単に説明します。

## 解決策の課題 1:SaaS AI による従業員の生産性

### ビジネスの問題

営業、マーケティング、人事、財務に携わる従業員が、メールの下書き、会議の要約、内部ドキュメントの検索などの日常的なタスクにかなりの時間を費やしていると報告しています。

### 選択したソリューション:Microsoft 365 Copilot (SaaS)

Contoso は、使い慣れた Microsoft 365 アプリケーションに直接統合されたターンキー SaaS 生成 AI ソリューションである **Microsoft 365 Copilot** を採用して、AI への取り組みを開始します。

SaaS を使用する理由

- 迅速な結果:インフラストラクチャまたはモデルの管理は必要ありません。

- スキルの障壁が低い:従業員は、既に知っているツール内で AI を使用します。

- 組み込みのガバナンス:アクセスは、Microsoft Entra ID と既存の Microsoft 365 アクセス許可で管理されます。

- 既定では、責任ある AI:データは Microsoft 365 テナント内に残り、既存のコンプライアンス コントロールを尊重します。

### 結果

数週間で、従業員から、ドキュメント作成の高速化、会議の概要の改善、情報の検索に費やす時間の短縮など、目に見える生産性の向上が報告されました。 この早期の成功で、より広範な AI 導入に対する組織の信頼と勢いが生まれます。

## 解決策の課題 2:PaaS AI を使用した顧客向けチャット

### ビジネスの問題

Tailwind Traders の eコマース サイトには、製品、返品、注文状態に関するお客様の質問に回答できるインテリジェントでスケーラブルなチャット機能がありません。

### 選択したソリューション:Azure AI Foundry と Azure OpenAI (PaaS)

顧客向けのカスタム ソリューションをサポートするために、Tailwind Traders は、**Azure AI Foundry と Azure OpenAI モデルを使用して、検索拡張生成 (RAG) チャットボット**をビルドします。

PaaS を使用する理由

- カスタマイズ:チャットボットは、Tailwind Traders の製品カタログ、FAQ、ポリシー ドキュメントに基づいています。

- マネージド操作:Azure により、スケーリング、可用性、更新プログラムの適用が処理されます。

- セキュリティ: モデルとデータは、Tailwind Traders の Azure サブスクリプション内にデプロイされます。

- 柔軟性:生成と非生成の両方の AI API をサポートします。

アーキテクチャの特長

- 自然言語対応のための Azure OpenAI。

- 承認済みの製品とポリシー データに基づいて応答する Azure AI Search。

- チャット アプリケーションをホストする Azure Container Apps。

- 安全なデータ フローのためのプライベート エンドポイントと制御されたネットワーク アクセス。

### 結果

お客様はより迅速で正確な応答を受け取り、満足度が向上し、コンバージョン率が向上します。しかも、すべてにおいて機密データをパブリック エンドポイントに公開する必要はありません。

## 解決策の課題 3:機械学習を使用したインベントリ予測

### ビジネスの問題

インベントリ レベルは、履歴データに基づく静的ルールを使用して管理されるため、非効率と収益機会の損失が発生します。

### 選択したソリューション:Azure Machine Learning (PaaS)

Tailwind Traders は **Azure Machine Learning** を選択して、過去の売上、季節的傾向、プロモーション データを使用して需要を予測する予測モデルを構築します。

Azure Machine Learning を使用する理由

- エンドツーエンドの ML ライフサイクル:データ準備、トレーニング、デプロイ、監視。

- 非生成 AI のサポート:予測や予報のユース ケースに最適です。

- エンタープライズ対応:ロールベースのアクセス、バージョン管理されたモデル、および実験の追跡。

- 統合:既存の Azure データ サービスと連携します。

### 結果

組織は、より正確な予測を実現し、過剰な在庫を削減できます。
